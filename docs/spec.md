# 機能仕様書 - LED Pattern Editor v0.1

更新日：2026-06-23

---

## 1. プロジェクト概要

WS2812B等のフルカラーLEDストリップ向けに、点灯パターンをブラウザ上で視覚的に作成・シミュレーション・転送するGUIツール。
ターゲットユーザーは開発者本人。将来的に小中学生向け講習での利用も視野に入れ、UIラベル・説明文は平易な日本語とする。

## 5. モジュール詳細

### 5.1 レイアウトエディタ

LEDの物理的な配置を定義する。

**機能**
- LED個数の設定（最大256個）
- キャンバス上にLEDをドラッグ＆ドロップで自由配置
- 配置プリセット：直線、蛇行（ジグザグ）、格子、円形
- LED番号（0始まり）と物理座標のマッピングを保持
- 整列・リセット機能

**データ構造**
```json
{
  "layout": {
    "led_count": 10,
    "leds": [
      { "id": 0, "x": 50,  "y": 100 },
      { "id": 1, "x": 100, "y": 100 }
    ]
  }
}
```

---

### 5.2 タイムラインエディタ

点灯パターンを時系列で定義する。

**画面構成**
- 横軸：時間（フレーム番号 / 秒表示切替）
- 縦軸：LED番号（0〜N-1）
- セル：各（LED × フレーム）の色をクリックで設定

**機能**
- セルクリックでカラーピッカー表示・色設定
- 範囲選択して一括塗り
- フレームのコピー・ペースト・削除・挿入
- ループ設定（ON/OFF）
- FPS設定（1〜60、デフォルト：10）
- タイムラインのスクロール

**デフォルト値（新規プロジェクト作成時）**
- LED数：10個（直線配置）
- フレーム数：10フレーム
- FPS：10
- ループ：ON
- 全セルの初期色：黒（R=0, G=0, B=0）

**フレームデータの記述方式**
- `frames` 配列は全フレーム分を完全記述する（差分ではない）
- 存在しないフレームは黒として扱う（将来的な最適化のため）

**範囲選択の操作方法**
- ドラッグ（マウスボタン押下のまま移動）で矩形選択
- 実装コスト次第でShift+クリックによる範囲選択に変更可

**将来拡張（キーフレーム型）**
- 指定フレームにキーフレームを置き、中間を線形補間（各チャンネルRGB独立）
- 補間モード：線形 / ステップ（瞬時切り替え）

---

### 5.3 シミュレーター

レイアウトエディタで定義した配置上でパターンをプレビューする。

**機能**
- ループ再生
- 再生速度変更：0.25× / 0.5× / 1× / 2× / 4×
- 一時停止・フレーム単位手動送り
- 現在フレーム番号・経過時間の表示
- LED表示：円形ドット（点灯：指定色、消灯：暗いグレー）

**LED表示の色値**
- 点灯時：パターンで指定した RGB 色
- 消灯時：`rgb(10, 10, 10)`（暗いグレー）

**未実装（v1.0スコープ外）**
- ガンマ補正（将来オプションとして追加可能な設計にする）

---

### 5.4 転送・エクスポート

**v1.0（Phase 2）：ファイルダウンロード**
- バイナリファイルを生成してブラウザからダウンロード（`.led` 拡張子）
- esptoolでの書き込み手順をUIに表示

**プロジェクト保存・読み込み**
- JSON形式（`.ledproj`）でプロジェクト全体を保存・読み込み
- 読み込み時にファイル形式のバリデーションを実施し、不正データはエラーメッセージを表示して中断する
- LED数変更時：既存ID のデータは引き継ぎ、増加分は黒（0, 0, 0）で初期化、削除分は破棄

## 7. データフォーマット

### 7.1 プロジェクトファイル（`.ledproj`）

```json
{
  "version": 1,
  "name": "サンプルパターン",
  "layout": {
    "led_count": 10,
    "leds": [
      { "id": 0, "x": 50, "y": 100 }
    ]
  },
  "pattern": {
    "fps": 10,
    "loop": true,
    "frame_count": 30,
    "frames": [
      {
        "frame": 0,
        "leds": [
          { "id": 0, "r": 0, "g": 0, "b": 255 },
          { "id": 1, "r": 0, "g": 0, "b": 0 }
        ]
      }
    ]
  }
}
```

### 7.2 転送バイナリフォーマット（`.led`）

#### ヘッダ（16バイト）

| オフセット | サイズ | 内容 |
|---|---|---|
| 0x00 | 2B | マジックナンバー `0x4C45`（"LE"） |
| 0x02 | 1B | バージョン `0x01` |
| 0x03 | 1B | フラグ（bit0: ループ有無） |
| 0x04 | 2B | LED数（リトルエンディアン） |
| 0x06 | 2B | フレーム数（リトルエンディアン） |
| 0x08 | 1B | FPS（1〜60） |
| 0x09 | 7B | 予約（`0x00`で埋める） |

#### フレームデータ（ヘッダ直後〜）

各フレーム = LED数 × 3バイト（**GRB順**、WS2812B対応）

```
[Frame 0] [LED0: G R B] [LED1: G R B] ... [LED(N-1): G R B]
[Frame 1] ...
```

**注意：内部データはRGB、バイナリ出力時にGRBへ変換する**

#### 総サイズ

```
16 + (LED数 × 3 × フレーム数) バイト
例：LED 10個、30フレーム → 916 バイト
```
